# 文件传输安全约束

## 规则

### 1. 禁止传输的内容
以下内容**禁止**通过网络传输：
- 个人隐私信息（身份证号、手机号、银行卡号、地址等）
- 认证凭据（API Keys、Tokens、密码、私钥等）
- 配置文件中的敏感信息

### 2. 允许传输的内容
- 产品图片、产品文档
- 代码文件（不含凭据）
- 非敏感的配置文件

### 3. 特殊场景：凭据传输
若必须传输认证凭据，必须满足：
- **必须**由人类审批同意
- 凭据必须脱敏处理（只显示前3位和后3位，如 `ghp_***...***`）
- 记录传输日志

### 4. 检查清单

传输前检查：
```
□ 不含个人隐私信息
□ 不含认证凭据（API Key、Token、密码）
□ 如含凭据，已获得人类审批
```

## 执行

- GitHub 凭据存储在 `/root/.openclaw/credentials/`
- 传输前使用 `grep -r "key\|token\|password" <目录>` 检查
